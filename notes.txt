1. Trigger and Configuration
The flow needs to run once every 24 hours.

Step: Recurrence (Schedule)

Interval: 1

Frequency: Day

Step: Initialize variable

Name: TargetDate

Type: String

Value: addDays(utcNow(), -7, 'yyyy-MM-dd')

This calculation finds the exact date from 7 days ago to compare against the "Created" field.

2. Retrieve Relevant Items
Instead of getting all items and using a "Condition" step (which is slow), we filter the data at the source.

Step: Get items (SharePoint)

Site Address: [Your Site URL]

List Name: Policy Draft Review

Filter Query: Created ge '@{variables('TargetDate')}T00:00:00Z' and Created le '@{variables('TargetDate')}T23:59:59Z' and Phase eq 'Phase 1' and (Status eq 'In Review' or Status eq 'Waiting on Update')

Top Count: (Optional) Set to 5000 if you have a massive list.

3. Process Results and Call Child Flow
Power Automate will automatically wrap these steps in an Apply to each loop when you reference the dynamic content from the previous step.

Step: Apply to each

Select an output from previous steps: value (from Get items)

Inside the Loop:
Since your Reviewers and Owners fields are likely "Person" columns (allowing multiple selections), we need to extract their emails into a string.

Step: Select (Data Operations)

From: Reviewers (Dynamic Content)

Map: Enter Email on the right side and select the Reviewer Email dynamic content.

Step: Join (Data Operations)

From: Output of the Select step above.

Join with: ; (Semicolon)

Repeat the Select and Join steps for the Owners field.

Step: Run a Child Flow (Flow)

Child Flow: [Select your Email Reminder Flow]

Parameters: * Pass the Joined Reviewer Emails

Pass the Joined Owner Emails

Pass the Policy Title/ID (so the email knows which item it's for)
