1. Trigger and Date Calculation
Step: Recurrence (Daily)

Step: Initialize variable (TargetDate)

Value: addDays(utcNow(), -7, 'yyyy-MM-dd')

2. Get and Filter Phase 2 Items
We pull all items that are 7 days old and stuck in Phase 2.

Step: Get items (SharePoint)

Filter Query: Created ge '@{variables('TargetDate')}T00:00:00Z' and Created le '@{variables('TargetDate')}T23:59:59Z' and Phase eq 'Phase 2' and (Status eq 'In Review' or Status eq 'Waiting on Update')

3. Extract Unique Reviewers
Even though you mentioned the reviewers are the same across records, some records might have different people in the "ReviewedBy" column. We need a clean list of people who still have work to do.

Step: Initialize variable (ReviewerArray)

Type: Array

Value: [] (Empty)

Step: Apply to each (Iterate through the Get items results)

Action Inside: Append to array variable (ReviewerArray)

Value: The Reviewers dynamic content from SharePoint.

Step: Compose (UniqueReviewers)

Inputs: union(variables('ReviewerArray'), variables('ReviewerArray'))

The union() function with itself is a clever "Power Automate hack" to remove all duplicate entries from an array.

4. Notify Each Reviewer
Now we loop through the unique list of people and send them their specific link.

Step: Apply to each (EachUniqueReviewer)

Select an output: Output of the UniqueReviewers Compose step.

Inside the Loop:
We need to check if this specific reviewer has already reviewed all the documents. If their name is in the "ReviewedBy" column for a specific record, they don't need a reminder for that record. However, your requirement is a single email pointing to their "My Active Items" view.

Step: Send an email (V2)

To: Items('EachUniqueReviewer')?['Email']

Subject: Action Required: Phase 2 Policy Draft Reviews

Body: > "Hello,

You have pending Phase 2 Policy reviews that reached the 7-day mark today. Please visit your personalized dashboard to complete these:

[Link to SharePoint View]"
